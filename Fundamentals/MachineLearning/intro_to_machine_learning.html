Introduction#

Machine learning (การเรียนรู้ของเครื่อง) คือสาขาหนึ่งของปัญญาประดิษฐ์ (Artificial Intelligence, AI) ที่เน้นการพัฒนาและออกแบบวิธีการให้เครื่องคอมพิวเตอร์สามารถเรียนรู้จากข้อมูลโดยไม่จำเป็นต้องระบุโครงสร้างหรือกฎกำหนดอย่างชัดเจน เครื่องคอมพิวเตอร์จะสามารถวิเคราะห์และสกัดความรู้หรือแบบแผนที่อยู่ภายในชุดข้อมูลขนาดใหญ่ ทำนายผลลัพธ์ หรือทำงานอื่นได้ โดยจะต้องอาศัยอัลกอริทึม (Algorithms) ต่างๆ ในการเรียนรู้

AI-Machine Learning-Deep Learning ที่มา

แผนภาพข้างต้นแสดงถึงวิวัฒนาการ AI ซึ่งก็คือความฉลาดเทียมที่สร้างขึ้นให้กับสิ่งไม่มีชีวิต โดยมี Machine Learning หรือการเรียนรู้ของเครื่อง จัดเป็นส่วนหนึ่งและเปรียบเหมือนสมองของ AI และส่วน Deep Learning (การเรียนรู้เชิงลึก) จัดเป็นการเรียนรู้ของเครื่องด้วยอัลกอริทึมโครงข่ายประสาทเทียม (Artificial Neural Networks, ANNs) ซึ่งเลียนแบบการทำงานของระบบประสาทของมนุษย์

การจำแนก Machine learning ตามการเรียนรู้ของข้อมูล#

เราสามารถจำแนกประเภทของ Machine learning ตามการเรียนรู้ของข้อมูลได้เป็น

  • Supervised Learning (การเรียนรู้แบบมีผู้สอน) เป็นการเรียนรู้ที่ต้องอาศัยชุดข้อมูลและชุดคำตอบของข้อมูล (labeled data) ในการฝึกฝนโมเดล เพื่อให้เกิดการเรียนรู้และสร้างรูปแบบในการจำแนกหรือทำนายผลลัพธ์ของชุดข้อมูลที่ต้องการศึกษา โมเดลที่เป็นที่นิยมใช้สำหรับ Supervised Learning ได้แก่ Linear Regression, Logistic Regression, Support Vector Machine (SVM), Decision Tree, Random Forest, Neural Networks เป็นต้น โดยทั่วไปในการทำนายผลลัพธ์ของการเรียนรู้ชนิดนี้สามารถแบ่งได้เป็น Classification(การจำแนกหมวดหมู่): เป็นการทำนายผลลัพธ์ที่มีลักษณะเป็นค่าแบบกลุ่มหรือประเภท โดยตัวอย่างที่นิยมใช้ เช่น การจำแนกอีเมล์เป็นอีเมล์ที่เป็นสแปมหรือไม่เป็นสแปม หรือการจำแนกภาพถ่ายเป็นภาพถ่ายของคนหรือสัตว์ และ Regression (การทำนายค่า): เป็นการทำนายผลลัพธ์ที่มีลักษณะเป็นค่าต่อเนื่องหรือตัวเลข โดยตัวอย่างที่นิยมใช้ เช่น การทำนายราคาของบ้าน การทำนายยอดขายสินค้าในอนาคต เป็นต้น

  • Unsupervised Learning (การเรียนรู้แบบไม่มีผู้สอน): ลักษณะของการเรียนรู้ประเภทนี้อาศัยชุดของข้อมูลซึ่งไม่มีการกำหนดคำตอบหรือผลลัพธ์ให้ โมเดลจะเรียนรู้โดยการสำรวจโครงสร้าง หาความสัมพันธ์โครงสร้างข้อมูล ตัวอย่างโมเดลที่เป็นที่รู้จักของ Unsupervised Learning ได้แก่ Clustering (การจัดกลุ่ม), Dimensionality Reduction (การลดมิติของข้อมูล เพื่อลดความซับซ้อนก่อนนำไปใช้ต่อ หรือเพื่อแสดงผลในรูปกราฟอ่านได้ง่ายขึ้น), Principal Component Analysis (PCA ซึ่งเราได้เคยพูดถึงในบทเรียนก่อนหน้านี้แล้ว) เป็นต้น

  • Reinforcement Learning (การเรียนรู้แบบเสริมกำลัง): เป็นการเรียนรู้ที่โมเดลพัฒนาตนเองขึ้น จากการลองผิดลองถูกในสถานการณ์ต่างๆ และจากการสังเกตสิ่งแวดล้อมรอบตัว ภายใต้เงื่อนไขของการได้รับรางวัลเมื่อโมเดลนั้นทำได้ถูกหรือตอบถูกนั่นเอง ตัวอย่างของ Reinforcement Learning ที่ถูกนำมาประยุกต์ใช้ในเกมส์ เช่น AlphaGo เป็น AI ที่เรียนรู้และฝึกหัดจากข้อมูลเกมส์การแข่งขันโกะจำนวนมหาศาลของผู้เล่นต่างๆ รวมถึงเกมส์ที่เล่นกับตัวเอง จนมีความสามารถเล่นเกมโกะให้ชนะผู้เล่นระดับโลกได้ ในส่วนของ Unsupervised Learning และ Reinforcement Learning หากสนใจ สามารถศึกษาข้อมูลเพิ่มเติมได้จาก https://towardsdatascience.com/ และ https://scikit-learn.org/

ภาพแสดงการจำแนก Machine Learning ตามการเรียนรู้ของข้อมูล: Supervised Learning, Unsupervised Learning และ Reinforcement Learning ที่มา

กระบวนการสร้างแบบจำลอง Machine Learning#

แผนภาพแสดงกระบวนการสร้างแบบจำลอง Machine Learning ที่มา

แผนภาพด้านบนแสดงกระบวนการสร้างแบบจำลอง Machine Learning เราจะสังเกตุเห็นได้ว่า ขั้นตอนการเก็บข้อมูล และการเตรียมข้อมูลนั้นมีความสำคัญอย่างมาก ไม่แพ้ขั้นตอนการสร้างโมเดลเลยทีเดียว ซึ่งเราอาจจะต้องใช้ความละเอียดรอบคอบและเวลาพอสมควรกับขั้นตอนนี้ ต่อไปเราจะมาลงรายละเอียดของแต่ละขั้นกัน

  1. การเก็บรวบรวมข้อมูล (Data Collection): ข้อมูลเหล่านี้อาจเป็นข้อมูลที่ไร้โครงสร้าง (unstructered data) เช่น ข้อความ (free text) รูปภาพ (images) ค่าตัวเลข (numeric) หรือข้อมูลที่มีโครงสร้าง (structured data) โดยข้อมูลที่มีโครงสร้างมักถูกจัดเก็บในรูปตาราง หรือ text ที่มีลักษณะโครงสร้างแน่นอนและสามารถนำไปใช้ได้ง่ายเช่น tsv และ csv

  2. การตรวจสอบและจัดเตรียมข้อมูล (Data Preprocessing): การตรวจสอบและจัดเตรียมข้อมูลให้พร้อมใช้งาน ประกอบด้วย 2.1 การทำความสะอาดข้อมูล (Data Cleaning) ด้วยการจัดการข้อมูลที่หายไปโดยการเติมข้อมูล (Data Imputation) หรือการละทิ้งข้อมูลที่เกี่ยวข้อง (Drop) 2.2 การแปลงข้อมูลให้เป็นรูปแบบที่เหมาะสม (Transformation) ด้วยการปรับรูปแบบข้อมูลให้อยู่ในรูป wide หรือ long format เพื่อให้คอมพิวเตอร์สามารถนำไปใช้งานได้ง่าย 2.3 การรวมข้อมูล (Data Integration) โดยการนำข้อมูลที่เกี่ยวข้องทั้งหมด มารวบรวมให้เป็นข้อมูลชุดเดียวกันโดยทำการเชื่อมข้อมูลของ บุคคลหรือวัตถุสิ่งเดียวกันให้อยู่ใน entry เดียวกัน หรือสามารถอ้างอิงถึงกันได้ง่าย

  3. การแยกแยะลักษณะ (Feature Extraction): ขั้นตอนนี้จะแปลงข้อมูลให้อยู่ในลักษณะที่เอาไปใช้งานได้ใน machine learning เช่น การแปลงข้อมูล text หรือรูปภาพ ให้อยู่ในรูปแบบตัวเลข อาเรย์ของตัวเลข (numeric array or vector) หรือแม้กระทั่ง bytes ตามสมมติฐานการใช้งาน และความเหมาะสมต่อโมเดลที่เลือกใช้

  4. การเลือกแบบจำลอง (Model Selection): คือการพิจารณาความเหมาะสมของปัญหาและข้อมูลที่มีอยู่ โดยเลือกแบบจำลองที่เหมาะสมต่อผลลัพท์ที่ต้องการ เช่น แบบจำลองการทำนายค่า (regression) เหมาะสำหรับปัญหาที่ต้องการผลลัพธ์แสดงค่าตัวเลข(หรือชุดของตัวเลข)ที่มีความต่อเนื่อง หรือมีสมบัติความเป็นลำดับขั้น ordinal แบบจำลองการจำแนกประเภท (classification) เหมาะสำหรับปัญหาที่ต้องการจำแนกกลุ่มหรือชุดของข้อมูลตามข้อมูลอ้างอิง (label) อื่นๆ เช่น ชื่อเรียก และตัวอย่างสุดท้ายคือแบบจำลองการจัดกลุ่มข้อมูล (clustering) ซึ่งเหมาะสมกับปัญหาในการวิเคราะห์หากลุ่มย่อยที่ไม่ได้นิยามหรือจำกัดความไว้ก่อน ซึ่งเป็นปัญหาแบบ unsupervised คือไม่ใช้ความรู้ใดๆที่มีอยู่เดิมนอกจากข้อมูลที่เราใช้ในแบบจำลอง

  5. การฝึกแบบจำลอง (Model Training): ในกระบวนการนี้จะเป็นการให้ข้อมูลที่เตรียมแก่แบบจำลองที่เราเลือก เพื่อให้แบบจำลองใช้ฝึกฝนและเรียนรู้วิธีการที่จะแก้ปัญหาให้ได้ตามข้อมูลที่สอน

  6. การประเมินแบบจำลอง (Model Evaluation): เมื่อแบบจำลองถูกฝึกแล้ว จะต้องทำการประเมินผลประสิทธิภาพของแบบจำลอง โดยใช้ชุดข้อมูลทดสอบที่แยกไว้เพื่อประเมินค่าเชิงสถิติ เช่น ค่าความถูกต้อง (accuracy) ค่าความแม่นยำ (precision) ค่าความไว (recall) หรือค่าความคลาดเคลื่อนเฉลี่ย (mean squared error) เป็นต้น

สิ่งที่กล่าวมาข้างต้น เป็นหลักการพื้นฐานที่เราสามารถนำมาใช้เพื่อเป็นแนวทางในการศึกษาเพื่อสร้างแบบจำลองที่ตรงตามความต้องการและแก้ปัญหาได้อย่างเหมาะสม มุมมองการตั้งคำถามหรือเป้าหมายในการสร้างแบบจำลองเองก็เป็นส่วนสำคัญที่จะทำให้เราสามารถใช้ machine learning ในการแก้ปัญหาได้อย่างมีประสิทธิภาพ นอกจากวิธีการพื้นฐานดังแสดงข้างต้นแล้วยังมีตัวอย่างการวิเคราะห์ปัญหารูปแบบอื่นๆเช่น https://developers.google.com/machine-learning/problem-framing ซึ่งแต่ละวิธีก็มีข้อดีและข้อด้อยต่างกัน และขึ้นกับความถนัดของผู้ใช้ด้วยเช่นกัน

#Model Evaluation การประเมินแบบจำลองหรือโมเดลเป็นหนึ่งในขั้นตอนที่สำคัญในการวัดประสิทธิภาพ (performance) ของโมเดลว่าดีเพียงพอต่อการนำไปใช้งานหรือไม่ กระบวนการนี้มีความสำคัญไม่น้อยไปกว่าการเลือกและการสร้างโมเดล ในการทำนายผลลัพธ์ที่มีลักษณะเป็นตัวเลขหรือค่าต่อเนื่องนั้น (regression) การวัดผลสามารถใช้ค่าเหล่านี้ซึ่งได้แก่ R-squared, Mean Absolute Error (MAE), Mean Squared Error (MSE), Root Mean Squared Error (RMSE) สามารถศึกษาเพิ่มเติมได้จาก https://scikit-learn.org/stable/supervised_learning.html ในเนื้อหาต่อไปนี้ เราจะมาเรียนรู้เกี่ยวกับการประเมินประสิทธิภาพของแบบจำลอง Machine Learning สำหรับปัญหาทาง Supervised learning: classification เนื่องจากพบเห็นได้บ่อยกว่ากับงานวิจัยทางด้านสมอง โดยสามารถพิจารณาได้จากค่าที่สำคัญต่างๆ ดังนี้ accuracy, confusion matrix, precision, recall, F1 score, ROC curve, AUC เป็นต้น

  • Confusion Matrix เป็นตารางที่แสดงผลลัพธ์จากการทำนายของ Machine Learning ในการแก้ปัญหา classification โดยที่ขนาดของตารางขึ้นอยู่กับจำนวนคลาสที่เราต้องการทำนายผลลัพธ์ เช่น สำหรับ 2 คลาสจะเป็นตารางขนาด 2 x 2 ถ้ามากกว่า 2 คลาส เราสามารถสร้างตาราง multiclass แบบ 3 x 3, 4 x 4 หรือ 5 x 5 ได้ แผนภาพด้านล่างแสดง Confusion Matrix (2 คลาส) โดยแต่ละช่องประกอบด้วย True Positives (TPs),True Negatives (TNs), False Positvies (FPs), และ False Negatives (FNs)

แผนภาพแสดง Confusion Matrix ที่มา

โดยที่

  • TPs: True Positives จำนวนกรณีที่เป็นจริงและทำนายว่าเป็นจริง

  • TNs: True Negatives จำนวนกรณีที่เป็นเท็จและทำนายว่าเป็นเท็จ

  • FPs: False Positives จำนวนกรณีที่เป็นเท็จแต่ทำนายว่าเป็นจริง

  • FNs: False Negatives จำนวนกรณีที่เป็นจริงแต่ทำนายว่าเป็นเท็จ

  • ค่าความถูกต้อง (Accuracy): เป็นค่าที่แสดงถึงความแม่นยำของโมเดลโดยรวม ใช้บอกว่าโมเดลสามารถทำนายได้ถูกต้องและตรงกับสิ่งที่เกิดขึ้นจริงเท่าไรเมื่อเทียบกับจำนวนการทำนายทั้งหมด โดยค่า accuracy จะมีค่าอยู่ระหว่าง 0-1 ยิ่งเข้าใกล้ 1 แปลว่าโมเดลเราทำนายผลได้ดีมาก \ \({Accuracy} = \frac{{TP} + {TN}} {{TP} + {TN} + {FP} + {FN}}\)

  • ค่าความแม่นยำ (Precision): เป็นค่าที่แสดงถึงความสามารถของโมเดลในการทำนายว่าได้แม่นยำแค่ไหนในคลาสที่สนใจ \ \(Precision = \frac{TP}{TP+FP}\)

  • ค่าความอ่อนไหวหรือค่าความไว (Sensitivity/Recall): แสดงถึงความสามารถของโมเดลในการทำนายได้อย่างถูกต้อง เทียบกับผลการทำนายที่ทั้งถูกและผิดของโมเดล \ \(Recall = \frac{TP}{TP+FN}\)

  • ส่วนค่าความจำเพาะ (Specificity) คือ สัดส่วนความถูกต้องของผลการทำนายในคลาสที่เราไม่ได้สนใจ (True Negative) โดยมีค่าเท่ากับ \(Specificity = \frac{TN}{TN+FP}\)

  • F1 Score: ค่าเฉลี่ยฮาร์โมนิกระหว่าง precision และ recall \ \(F1\text{-}Score = 2 \times \frac{Precision \times Recall}{Precision + Recall}\)

  • ROC Curve (Receiver Operating Characteristic Curve): กราฟที่แสดงความสามารถในการแยกแยะของโมเดลระหว่าง TPR: True Positive Rate (sensitivity) และ FPR: False Positive Rate (1-specificity) เป็นการพล็อตระหว่างค่าสัดส่วนที่ทำนายได้อย่างถูกต้อง กับค่าสัดส่วนที่ทำนายผิด (false positive) โดยที่ \(TPR = \frac{TP}{TP+FN}\), \ \(FPR = \frac{FP}{FP+TN}\) \

  • AUC (Area Under the ROC Curve): ค่าของพื้นที่ใต้เส้นโค้ง ROC ซึ่งสามารถใช้เป็นตัวบ่งชี้ของประสิทธิภาพของโมเดลในการแยกแยะ AUC มีค่าอยู่ระหว่าง 0-1 ยิ่งเข้าใกล้ 1 แปลว่าโมเดลในภาพรวมสามารถทำนายผลลัพธ์ได้ดีมาก

แผนภาพแสดง ROC curve ซึ่งค่าทางแกน X คือ False Positive Rate (TPR) และค่าทางแกน Y คือ True Positive Rate (FPR) ที่มา

การวัดผลประสิทธิภาพโมเดลด้วย ROC curve: เส้นประสีแดงแสดงประสิทธิภาพของโมเดลแบบทำนายสุ่ม เส้นทึบสีเขียวและเส้นทึบสีฟ้าที่อยู่เหนือเส้นประสีแดง แสดงถึงโมเดลที่มีประสิทธิภาพดี ในขณะที่ต่ำกว่าเส้นสีแดง แสดงถึงโมเดลมีประสิทธิภาพต่ำ จากเส้น ROC curves เราสามารถประมาณพื้นที่ใต้กราฟหรือ AUC (Area under the ROC curve) ของโมเดลที่เกิดจากการทำนายแบบสุ่ม (random classifier) ได้เท่ากับ 0.5 โมเดลที่มีประสิทธิภาพดีนั้นจะมีค่า AUC เข้าใกล้ 1 ซึ่งเส้น ROC curve จะเข้าใกล้ทางด้านมุมซ้ายมากที่สุด อย่างไรก็ตามในการใช้งานจริงเราควรจะต้องพิจารณาค่า accuracy, precision, recall, F1, confusion matrix, และ AUC ประกอบกัน ซึ่งเราสามารถเขียนโค้ด เรียกไลบรารี่และฟังก์ชันสำหรับการคำนวณค่าเหล่านี้ได้จาก scikit-learn

ตัวอย่างการใช้ Confusion Matrix#

สมมติผลการทดสอบชุดตรวจโควิดแบบพกพา (ATK) ของบริษัทหนึ่งเปรียบเทียบผลการทดสอบด้วย Real time RT-PCR ซึ่งเป็นวิธีมาตรฐานและแม่นยำ จากจำนวนผู้ทดสอบทั้งหมด 1,584 ราย มีค่าดังตาราง confusion matrix นี้

ผลในตารางแสดงว่า การใช้ชุดตรวจ ATK สามารถแยกผู้ติดเชื้อโควิดได้ถูกต้องจำนวน 1,069 ราย และผู้ไม่ติดเชื้อได้ถูกต้องจำนวน 405 ราย และพบว่ามีจำนวน 38 รายที่ชุดตรวจ ATK บอกว่าไม่มีเชื้อโควิด แต่จากการตรวจด้วย RT-PCR ยืนยันการติดเชื้อโควิด และที่น่าสนใจมีจำนวน 72 รายที่ ATK บอกว่าติดเชื้อแต่จากการตรวจด้วย RT-PCR พบว่าไม่ติดเชื้อ (ถือเป็น false alarm) ดังนั้นจะเห็นได้ว่าในกรณีของ false negatives (ผู้ป่วยติดเชื้อโควิด แต่ ATK ตรวจไม่พบเชื้อ) จะสร้างความเสียหายให้เกิดขึ้นได้ เนื่องจากการตรวจไม่เจอเชื้อ จะทำให้ผู้ป่วยสามารถไปแพร่เชื้อให้คนอื่นได้โดยไม่รู้ตัว

  • ลองคำนวณค่า accuracy, precision, specificity, sensitivity ของชุดตรวจ ATK ชนิดนี้ ด้วยสูตรการคำนวณที่อธิบายไว้ข้างต้น และอภิปรายถึงผลที่คำนวณได้ต่อประสิทธิภาพชุดตรวจ

  • จากค่าที่คำนวณ accuracy, specificity, precision, sensitivity ของชุดตรวจ ATK 1.ถ้าเราต้องการแยกผู้ป่วยโควิดให้ได้มากที่สุด สามารถดูได้จากค่าอะไร 2.ถ้าไม่ต้องการให้คนที่ไม่ได้ป่วยถูกทำนายผิด ควรพิจารณาจากค่าอะไร

หมายเหตุ: ทุกคนสามารถทบทวนเนื้อหาที่เกี่ยวข้องในส่วนของ Computational Modeling และ Neuroscience & AI ได้จาก Brain Buiding Blocks ในบทเรียนถัดไป เราจะมาเรียนรู้การสร้างโมเดล Machine learning สำหรับปัญหาทาง Supervised learning: classification ด้วย Logistic Regression, Support Vector Machine, Decision Tree, Random Forest, และ K-Nearest Neighbors บน datasets ที่สร้างขึ้น และช่วงท้ายเราจะได้ลองสร้างโมเดลที่ได้เรียนมาและประเมินประสิทธิภาพของโมเดลที่สร้างขึ้น กับปัญหางานวิจัยในการจำแนกอารมณ์ของคนจากคลื่นสัญญาณไฟฟ้าของสมอง (Electroencephalogram, EEG)
